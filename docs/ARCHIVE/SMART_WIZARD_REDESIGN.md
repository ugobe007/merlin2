## Implementation Notes

### Key Design Principles
1. **Progressive Disclosure**: Simple first, advanced behind toggles
2. **Visual Learning**: Use icons, diagrams, comparisons
3. **Instant Feedback**: Show cost/savings updates in real-time
4. **Plain Language**: No jargon unless necessary
5. **Guided Choices**: Recommend "best" options, explain why
6. **Exit Points**: Allow users to save and continue later

### Technical Requirements
- All calculations must work with partial data
- Save progress automatically (localStorage)
- Allow jumping between sections
- Mobile-responsive design
- Print-friendly output
- Share quote via link

### Validation
- Don't block progression on optional fields
- Show warnings, not errors
- Allow "I don't know" options
- Pre-fill intelligent defaults

### Help System
- Tooltip on every term
- "?" icons with explanations
- Live chat option
- "Talk to expert" button on every screen
