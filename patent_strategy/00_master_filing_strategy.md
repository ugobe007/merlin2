# Merlin Patent Filing Strategy

## Purpose

This memo organizes the Merlin intellectual property portfolio into four filing groups so that counsel can draft coherent applications rather than a long list of narrow software ideas. The guiding principle is to claim architecture, data flow, coupling, and enforcement patterns while preserving calibrated values and operational tuning as trade secrets.

## Filing Philosophy

### What to Claim

- Multi-stage decision flow and how stages constrain each other
- Dual-source and dual-destination data architectures
- Separation of layers that preserves auditability
- State-machine behavior that changes the user path or recommendation path
- Runtime and database enforcement mechanisms
- Platform expansion patterns that reuse shared backend logic across products

### What to Keep as Trade Secret

- Margin percentages
- Confidence weights
- Keyword lists and weight tables
- Exact pricing floors and ceilings
- Blend ratios
- Market reliability scores
- Vendor approval criteria
- Industry-specific sizing constants
- Review thresholds
- Recency decay curves

## Recommended Drafting Pattern

For each group, counsel should draft:

1. One broad independent system claim
2. One broad independent method claim
3. One independent non-transitory computer-readable medium claim if desired
4. Dependent claims that pull in the strongest implementation details without forcing disclosure of sensitive numeric values

## Preferred Timing

- Group 2 next
- Group 3 after Group 2
- Group 4 last, unless a specific module is driving customer demand, fundraising, or public disclosure risk

## Attorney Handoff Checklist

- Clean narrative summary for the selected group
- Architecture diagram showing data flow and state transitions
- List of named source files and core functions
- Screenshots or UI recordings demonstrating the live feature
- Dated evidence of launch, testing, or customer use
- Trade-secret carveout list identifying numbers and tuning parameters to omit
- Prior-art notes showing how Merlin differs from generic CPQ, generic ML pipelines, generic dashboards, and generic marketplaces

## Internal Notes

- Avoid describing the portfolio as "29 patents" in the filing process. Present the inventions as grouped platform families.
- Use the core family to support continuation strategy later, especially if counsel believes additional claims can be spun out from the first filing.
- If budget gets tight, prioritize the next filing order as Group 2, Group 3, then selected modules from Group 4 rather than the whole Group 4 bundle at once.
